# 🧩 拼圖戰力最佳化模擬器 (Puzzle Combat Power Optimization Simulator)

---

## 📝 專案提案 (Proposal Report)

### 💡 動機與目標
縱觀學生生涯我時常利用一些養成遊戲來打發時間，也因此接觸到了遊戲中的拼圖戰力系統。此系統以各種不同形狀的拼圖塊代表不同屬性增幅，要在 $n \times n$ 的拼圖格中放入拼圖塊，目標是讓戰力（即盤面覆蓋率）最大化。我注意到此系統普遍具備自動計算最佳拼法的功能，由此產生了強烈動機，想要動手復刻這個高效求解引擎，並深入研究其底層程式碼的運作原理。同時，我也希望將此想法藉由現代化的前端網頁 UI 介面呈現，讓使用者能實際拖放拼圖塊進行互動，或一鍵交由電腦進行極速求解。

### 📊 競品比較
除了原先遊戲內建的自動生成最佳方案的系統之外，與此專案有相似之處的競品包含以下兩類：
1. **商業遊戲內建系統**：如聖遺物系統、裝備戰力系統，提供最快的戰力提升方案。
2. **工業級裝箱求解軟體**：要求空間利用率以及數學精確度。

### ⚙️ 預期功能 
- **基礎功能**：以網頁方式實現本專案。包含拼圖塊資料庫存取、使用者手動拖曳嘗試、以及電腦一鍵求解最優盤面。核心演算法需支援拼圖塊的「旋轉」與「鏡像翻轉」組合。
- **進階功能**：建立可選取的拼圖庫，支援使用者挑選特定拼圖加入「待求解池」；允許自訂棋盤尺寸並設定演算法的時間上限。同時加入「障礙物模式」，模擬複雜的地形限制。

### 🛠️ 使用技術
- **演算核心**：`C++` (實作核心求解與空間策略演算法)。
- **前端介面**：`TypeScript` + `React` (負責畫面渲染、Drag & Drop 互動與非同步動畫展示)
- **跨語言整合**：`WebAssembly (WASM)` (使用 Emscripten 將 C++ 編譯為高效 Web 模組)
- **樣式架構**：`Tailwind CSS` (建構響應式、現代化電競風格 UI)
- **總結** ： 有結合了課堂中各個章節內所提到的內容進行編譯。雖然主軸是結合了我的選修 Web App 開發，但也有盡力結合了課堂中提及的技巧。

## 🧪 原型開發報告 (Prototype Report)

### 📈 目前進度
* **WASM 雙向橋樑**：完成 C++ 與 JavaScript 的通訊。前端能成功傳送陣列給 C++，C++ 也能流暢回傳代表棋盤最新狀態的一維線性陣列。
* **基礎邊界模型**：建立動態格點區域，精準判斷各式拼圖變換後是否越界或重疊，並支援在畫面上放置「黑洞障礙物」。
* **演算核心升級**：核心演算法由最初的暴力回溯法，到後續改為AI推薦為基於十字鏈表的 **DLX (Dancing Links X)** 演算法，並加入超時控制機制。

### ⚠️ 遇到的困難與解決方案
1. **數據格式轉換**：將 C++ 的指標轉換成 JS 陣列耗費了不少時間。最後決定全面採用「一維線性陣列」來模擬二維棋盤，大幅降低傳輸開銷。
2. **相對座標運算**：拼圖旋轉與翻轉時座標會偏離原點。特別寫了歸一化 (Normalize) 函數，確保每次變換後圖形都能貼齊左上角，解決邊界判定失準的問題。

### 下一步計畫
* **即時視覺回饋**：實作 Drag and Drop 系統，游標懸停時顯示半透明「幽靈拼圖」預覽位置，位置非法時實時給予紅色邊框警告。
* **動畫渲染**：為了讓運算結果有「運算感」，預計額外加入計時器 (setInterval) 動畫系統，讓盤面能一格一格依序顯示，也希望使用者模式手動拖曳拼圖塊時也可以有小動畫。

## 🚀 成果報告 (Final Report)

### 📋 專案說明
本專案成功打造出一個兼具互動性與運算效能的**「拼圖戰力最佳化模擬器」**。 在使用者模式下，使用者可以自行拖曳拼圖塊進入中央棋盤內，並且在左側也會顯示目前棋盤內的拼圖種類以及個數。系統模式下，會使用演算法計算使用者選取的拼圖塊所能達到的最佳覆蓋率，由於此問題在複雜情況下得出最佳解會需要很長時間，因此設立時限以防網頁完全卡死。使用者也可以在中央棋盤上方點擊上一步或者下一步，增加使用者的操作彈性，也可以點擊開啟障礙增加趣味性。另外，無論是使用者拖曳拼圖塊或者系統顯示拼圖塊擺放結果，都有設置動畫以增加動態感。拼圖塊擺放時也會隨機16種顏色來進行區分，以讓使用者能夠清楚了解拼圖塊的實際擺放和分界。

## 💻 安裝與啟動方式 (Getting Started)

### 1. 環境需求
- **Node.js**
- **Emscripten SDK (emsdk)**

### 2. 編譯 C++ 核心引擎 (WASM)
於專案根目錄執行：
```bash
emcc -O3 --bind engine.cpp -o public/puzzle_module.js \
  -s EXPORT_ES6=1 \
  -s MODULARIZE=1 \
  -s ALLOW_MEMORY_GROWTH=1 \
  -s EXPORTED_RUNTIME_METHODS="['ccall','cwrap']"
```
### 3. 啟動前端開發伺服器
- **npm install   # 安裝套件**
- **npm run dev   # 啟動開發伺服器**
