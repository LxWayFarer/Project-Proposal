# [專題名稱]：拼圖戰力最佳化模擬器

## Proposal Report
### 動機與目標：
   
   - 縱觀學生生涯我時常利用一些養成遊戲來打發時間，也因此接觸到了某遊戲的拼圖戰力系統。此系統以各種拼圖塊代表不同屬性增幅，要在n*n的拼圖格中放入拼圖塊，使得讓戰力最大化。我很快便注意到了此系統有自帶電腦計算最佳拼法的功能，由此產生了動機讓我想復刻這個功能，並且詳細瞭解其程式碼運作的原理。我也會嘗試將此想法藉由UI介面顯示出來，並且讓玩家能夠實際滑動拼圖塊放入棋盤或者進行其他操作。

### 競品比較
   - 除了原先遊戲內建的自動生成最佳方案的系統之外，與此專案有相似之處的競品包含以下兩類：
   - 1. 商業遊戲內建系統：如聖遺物系統、裝備戰力系統，提供最快的戰力提升方案。
   - 2. 工業級裝箱求解軟體：要求空間利用率以及數學精確度。

### 預期功能 
   - 基礎部分：希望能以網頁的方式實現此專案。功能包括拼圖塊的資料輸入/隨機生成拼圖塊的功能、令使用者滑動拼圖塊自行嘗試或者直接利用程式求解的功能等，但主要依然是生成放置拼圖塊的最優解，且會考慮拼圖塊旋轉後的組合。
   - 進階部分：建立一個可選取的拼圖庫，使用者可以手動挑選特定的拼圖組合；允許使用者自訂棋盤尺寸，並設定K階（階級越高，權重越高，拼圖塊越小）拼圖的積分權重。

### 使用的技術
   - C++ ：實作棋盤的最佳策略演算法。
   - 前端介面層 (UI/UX)：TypeScript + React：負責畫面渲染、拼圖拖放動畫、與 GitHub Pages 的整合，也可以考慮運用Canvas。
   - WebAssembly (WASM)工具：使用 Emscripten 將 C++ 編譯成 WASM 模組，讓 TypeScript 可以直接呼叫 C++ 的函數。
   - Tailwind CSS：快速建構響應式介面。

## Prototype Report

### 目前進度
   - 目前完成了以下部分：
   - 1. WASM 橋樑：完成 C++ 與 JavaScript 的通訊。JavaScript 能傳送棋盤大小給 C++，C++ 能回傳一個代表棋盤狀態的陣列。
   - 2. 基礎的「棋盤與拼圖」模型：目前已經生成一個n*n的格點區域，能夠判斷各種形狀的拼圖（限制每格必須連在一起），並且可以判斷拼圖塊是否有在邊界內。
   - 3. 初步的視覺回饋：網頁能根據 C++ 回傳的數據，把個別拼圖塊的格子染成對應的顏色，並且滑鼠懸停在棋盤或者拼圖塊上時，邊框會加粗。
   - 4. 最小可行性運算：嘗試3*3, 4*4下的最優解演算法，並且建立按鈕讓系統隨機生成合法形狀的拼圖塊。

### 遇到的困難
   - 遭遇的困難如下：
   - 1. 數據格式的轉換衝突，比如將 C++ 的指針 (Pointer) 或 Array Buffer 轉換成 JS 陣列就需要想一段時間。後來我聽取AI的建議先使用「線性陣列（一維陣列）」來代表二維棋盤，傳輸效率也比較高。
   - 2. 當拼圖塊包含「旋轉」或「翻轉」時，其相對座標會改變。由於涉及矩陣運算以及各種邊界判定，也需要花上很多時間修改錯誤。

### 下一步計畫
   - 1. 從「靜態模型」轉向「動態求解」：實作回溯演算法，讓它能自動尋找拼圖的組合，當然也必須把旋轉的可能性給包含進去。
   - 2. 戰力積分計算：建立一個計算引擎，當拼圖放入時，根據其階級（Level）即時計算並回傳當前總積分。
   - 3. 簡單的剪枝 (Pruning)：在遞迴中加入判斷。例如：如果剩餘的空格總數已經小於剩下拼圖的最小面積，就直接放棄該分支。
   - 4. 拼圖庫 UI：設計一個側邊欄（Sidebar），用來存放「待使用的拼圖塊」，並且處理處理拼圖從側邊欄「拖進」棋盤的座標轉換邏輯。
   - 5. 新增「一鍵求解」、「隨機生成拼圖」、「清除棋盤」以及「自訂n*n棋盤大小」的控制項。
