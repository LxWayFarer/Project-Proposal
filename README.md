## Proposal Report
### 題目名稱、動機與目標：
   - 題目名稱：遊戲戰力拼圖系統模擬
   
   - 題目詳細說明：
  假設一個n*n格的棋盤中，可以放下各種代表不同積分的拼圖塊。一階到K階的拼圖塊放入棋盤裡可得1分到K分，且在棋盤中分別佔有K個到1格的空間。給定拼圖塊的形狀（同時也能明顯知道它是幾階的拼圖塊）並嘗試利用演算法求出最佳策略以得到最高分數。
   
   - 動機：縱觀學生生涯我時常利用一些養成遊戲來打發時間，也因此接觸到了某遊戲的拼圖戰力系統。我很快便注意到了此系統有自帶電腦計算最佳拼法的功能，由此產生了動機讓我想復刻這個功能，並且詳細瞭解其程式碼運作的原理。我也會嘗試將此想法藉由UI介面顯示出來，並且讓玩家能夠實際滑動拼圖塊放入棋盤或者進行其他操作。

   - 預期功能：希望能以網頁的方式實現此專案。功能包括拼圖塊的資料輸入/隨機生成拼圖塊的功能、令使用者滑動拼圖塊自行嘗試或者直接利用程式求解的功能等。
  
### 使用的技術/語言：
   - C++ ：實作棋盤的最佳策略演算法。
   - 前端介面層 (UI/UX)：TypeScript + React：負責畫面渲染、拼圖拖放動畫、與 GitHub Pages 的整合。
   - WebAssembly (WASM)工具：使用 Emscripten 將 C++ 編譯成 WASM 模組，讓 TypeScript 可以直接呼叫 C++ 的函數。

### 初步時程規劃：
  #### 第一階段：架構設計與基礎邏輯 (Week 7 - Week 8)
  - 環境初始化：建立 GitHub Repository 並配置 Vite + React/Vue 開發環境。
  - 資料結構定義：設計 $n \times n$ 棋盤與 K 階拼圖塊的 JSON/Class 模型。
  - 驗證系統：實作拼圖放置的合法性檢查邏輯（碰撞偵測、邊界判定）。
  #### 第二階段：演算法實作與雛型開發 (Week 9 - Week 10)
  - 核心演算法：撰寫求取最高分最佳解的遞迴或回溯演算法。
  - 效能優化：實作剪枝（Pruning）邏輯，避免大規模棋盤運算過久。
  - 基礎 UI：渲染網頁棋盤，並能手動將拼圖塊放置到指定座標
  #### 第三階段：期中進度交付 (Week 11 - Prototype Report)
  - 更新 README：撰寫目前完成功能、技術困難點以及與課程內容（如資料結構）的關聯。
  - 功能檢核：確保 Web 版介面已能展示初步的拼圖模擬成果。
  #### 第四階段：功能強化與互動優化 (Week 12 - Week 13)
  - 互動性增強：加入拖放（Drag and Drop）功能與即時積分計算面板。
  - UI/UX 美化：優化視覺風格。
  #### 第五階段：最終測試與成果交付 (Week 14 - Week 15)
  - 錄製 Demo 影片：拍攝操作教學與演算法原理講解並上傳 YouTube。
  - 發布 Release final-report：完成所有 README 區段並整理代碼註釋。
  - 最終提交：於課程網頁繳交 GitHub Repo 與 Demo 連結。
